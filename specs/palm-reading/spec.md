# Feature Spec: Palm Reading (看手相)

## Overview

为 HYP 星座研究网站新增"看手相"功能模块。采用混合模式：默认趣味互动模式（离线可用），用户上传手掌照片后在图上叠加手掌线条参考图，自行点击识别感情线、智慧线、生命线、事业线，从内置手相学数据库获取解读；同时提供"AI 深度解读"按钮，用户填入自己的 Google Gemini API Key 后，将照片发送给 Gemini 多模态大模型进行真正的 AI 手相分析。

## Clarifications

### Session 2026-06-28

- Q: AI 深度解读发送照片前，应采用哪种授权方式？ → A: 仅展示静态隐私说明，不额外确认
- Q: 上传照片的左右手与方向应如何处理？ → A: 仅支持右手掌、手指朝上的标准照片
- Q: 照片缩放应支持哪些交互方式？ → A: 仅支持桌面端鼠标滚轮
- Q: 系统是否需要自动检查照片确实为“右手掌、手指朝上”？ → A: 不自动检查，仅提示用户遵循要求
- Q: “生命线”及 AI 解读应如何处理健康相关内容？ → A: 允许传统健康与寿命解读，但附免责声明

## 定位

娱乐为主，明确告知用户结果仅供参考。

## User Stories

### US1: 手掌照片上传与展示 (P1)

**As a** 访客，**I want to** 上传我的手掌照片并在页面上展示，**so that** 我可以在照片上进行手相分析。

**Acceptance Criteria:**
- 导航栏新增"手相"入口，路由 `#/palm`
- 页面中央有一个上传区域，带手掌图标和"上传手掌照片"提示文字
- 上传区明确提示仅支持右手掌、掌心朝向镜头且手指朝上的标准照片
- 系统不自动识别或校验左右手与照片方向，也不因方向问题阻止用户继续
- 支持点击上传和拖拽上传两种方式
- 支持 JPG/PNG/WebP 格式，文件大小限制 10MB
- 上传后照片显示在 Canvas 画布上，可旋转（左转/右转 90°）；仅桌面端支持鼠标滚轮缩放，移动端自动适配且不提供手动缩放
- 上传区域下方有"重新上传"按钮
- 照片仅在本地浏览器中处理，不上传到任何服务器（AI 模式除外）
- 页面底部注明"本功能仅供娱乐参考，不构成任何专业建议"

### US2: 趣味互动线条识别 (P1)

**As a** 访客，**I want to** 在上传的手掌照片上叠加线条参考图，点击识别我的掌纹线条，**so that** 我可以了解各条手相线的含义。

**Acceptance Criteria:**
- 符合“右手掌、掌心朝向镜头、手指朝上”要求的照片上传后，Canvas 上叠加半透明的手掌线条参考图，标注四条主线：感情线、智慧线、生命线、事业线
- 每条线有标注文字和序号（① 感情线 ② 智慧线 ③ 生命线 ④ 事业线）
- 用户点击某条线后，弹出该线条的解读面板，显示：
  - 线条名称（中英文）
  - 位置说明
  - 该线条代表的含义领域（感情线=情感、智慧线=思维、生命线=传统健康/活力/寿命解读、事业线=事业/命运）
  - 随机展示该线条的一种解读变体（如线条长短、深浅、弯曲度的含义）
- 用户可以逐一点击四条线，收集完整的"手相报告"
- 收集完四条线后，显示"手相报告"总结面板
- 提供"重新分析"按钮重置选择
- 所有解读从内置 JSON 数据库中读取，完全离线

### US3: AI 深度解读 (P2)

**As a** 访客，**I want to** 填入我的 Gemini API Key，让 AI 分析我的手掌照片，**so that** 我可以获得更个性化、更详细的手相解读。

**Acceptance Criteria:**
- 页面上方有"AI 深度解读"按钮和"设置 API Key"入口
- 点击"设置 API Key"弹出输入框，用户填入 Google Gemini API Key
- API Key 存储在 localStorage（`hyp_palm_apikey`），页面刷新后保留
- 提供 API Key 清除按钮
- 填入 Key 后，"AI 深度解读"按钮变为可用状态
- AI 区域常驻展示照片将直接发送至 Google Gemini API 的静态隐私说明；发起分析时不额外弹出确认
- 点击"AI 深度解读"后：
  - 将手掌照片转为 Base64 发送到 Gemini API（`gemini-2.5-flash` 稳定模型）
  - 请求提示词包含手相学分析指令（分析四条主线、提供中文解读、注明娱乐性质）
  - 允许生成传统手相中的健康与寿命解读，但结果必须注明仅供娱乐参考、不构成医疗建议
  - 显示加载动画"AI 正在分析手相..."
  - API 返回后，以格式化方式展示 AI 解读结果
  - 如果请求失败（Key 无效、网络错误等），显示友好的错误提示
- AI 结果区域支持"复制结果"和"重新分析"操作
- 明确标注"AI 解读由 Google Gemini 生成，仅供娱乐参考"

### US4: 手相知识百科 (P3)

**As a** 访客，**I want to** 浏览手相学知识库，**so that** 我可以学习手掌各线条的含义和基础知识。

**Acceptance Criteria:**
- 手相页面底部有"手相知识"折叠区域
- 展开后显示四条主线的图解和详细说明
- 每条线包含：位置图示、基本含义、常见变体解读（长短/深浅/弯曲/分支/断裂等）
- 支持点击切换不同线条查看
- 内容从内置数据库读取

## Data Model

### PalmLine (内置 JSON, `data/palm-reading.json`)

| Field | Type | Description |
|-------|------|-------------|
| id | string | 线条ID: "heart" / "head" / "life" / "fate" |
| name_cn | string | 中文名: "感情线" |
| name_en | string | 英文名: "Heart Line" |
| area | string | 位置说明 |
| domain | string | 代表领域: "情感与人际关系" |
| description | string | 基本描述 |
| interpretations | object[] | 解读变体数组 |
| interpretations[].trait | string | 特征: "长" / "短" / "弯曲" / "笔直" / "分叉" / "断裂" |
| interpretations[].meaning | string | 该特征对应的解读文字 |
| overlay | object | Canvas 叠加层坐标（归一化 0-1）|
| overlay.points | number[][] | 线条路径点 [[x,y],...] |
| overlay.labelPos | number[] | 标签位置 [x,y] |

### User Settings (localStorage)

| Key | Type | Description |
|-----|------|-------------|
| hyp_palm_apikey | string | 用户填入的 Gemini API Key |

## Non-Functional Requirements

- **隐私安全**: 照片仅在浏览器本地处理；AI 模式下照片 Base64 直接发送给 Gemini API，不经过其他第三方；AI 区域常驻展示该传输说明，不设置额外发送确认
- **离线优先**: 趣味互动模式完全离线可用，不依赖任何网络资源
- **性能**: 照片加载和 Canvas 渲染在 1 秒内完成；AI 请求超时设置为 30 秒
- **兼容性**: 与现有星座网站集成，共享导航栏和路由系统
- **文件限制**: 照片最大 10MB，超出时显示提示
- **健康免责声明**: 离线解读与 AI 解读可以包含传统健康及寿命说法，但必须紧邻结果展示"仅供娱乐参考，不构成医疗建议"
